定宽与自适应

float+margin

  • 优点： 兼容性较好

float+margin+(fix)

  • 优点： 兼容性较好
  • 缺点： 需要增加HTML结构

在right外套上一层fix

overflow(BFC模型)

  • 优点：兼容性较好，只须设置left与right

借助BFC清除float带来的环绕效果，进一步了解BFC

table

  • 优点： 无须考虑left的情况，只须设置right
  • 缺点： 由于flex,兼容性相对差一些。

table-layout默认为auto（自动表格布局）。还有一个值fixed（固定表格布局），该算法中表格的内容并不会影响单元格的宽度，只能影响高度，所以页面在下载过程中，表格不需要不断重绘，速度也就更加快了。另外该算法中，如果表格不指定宽度，各列的宽度将平均分配，利用这个特点可以实现等分布局（后面将会讲到）。

flex

  • 优点： 无须考虑left的情况，只须设置right
  • 缺点： 由于flex,兼容性相对差一些。

不定宽与自适应

有了上面的定宽与自适应思路，不定宽就容易了。如果CSS中设置了硬参数关系到left的宽度，就不能实现不定宽与自适应布局。
因此，经过排除有以下几种(HTML结构不变，随意设置.left宽度即可[模拟不定宽])

overflow

因为html嵌入了style(width)代码，不定宽无须涉及宽度，所以CSS样式与上面定宽自适应布局使用overflow代码一样

table

flex

等分

由公式可知，只要父容器增加一个间距宽度，那么总宽度就相当于每个元素增加一个间距。只要每个元素包含间距进行平分就能实现需求。

实现难点：

  • 如何让父容器增加一个间距
  • 如何在平分过程包含间距(25%包含间距)

下面假设间距为20px

margin

  • 优点：兼容性较好
  • 缺点：需要根据需求更改宽度的比例。(比如3个，宽度就需要重新设置)

table

table可以实现等分，如果考虑间距，就需要在结构上进行修改。
当设置了父容器宽度100%后，就不能像上一个方案增加一个间距宽度。因此，通过在外面加一个div，让外层div增加一个间距宽度。

  • 优点： 无须考虑子容器个数，能实现自动平分
  • 缺点： 需要修改HTML结构

flex

  • 优点： 无须考虑子容器个数，能实现自动平分
  • 缺点： 由于flex,兼容性相对差一些。
